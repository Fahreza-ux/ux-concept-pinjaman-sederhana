# Problem Statement

## Context
Berdasarkan observasi terhadap 5 aplikasi pinjaman existing, ditemukan masalah umum dalam user experience.

## Main Problems Identified

### 1. Complexity Anxiety
**Observation**: User merasa overwhelmed dengan form yang panjang dan istilah teknis.

**Evidence**:
- Form pengajuan 15+ field sekaligus
- Istilah seperti "bunga efektif", "tenor", "plafon" tidak dijelaskan
- Tidak ada progress indicator

### 2. Lack of Transparency
**Observation**: User tidak tahu status pengajuan dan estimasi waktu.

**Evidence**:
- Status hanya "diproses" tanpa detail
- Tidak ada estimasi kapan akan disetujui
- Customer service sulit dihubungi

### 3. Fear of Commitment
**Observation**: User takut salah pilih jumlah pinjaman dan tenor.

**Evidence**:
- Tidak bisa simulate different scenarios
- Tidak jelas total biaya yang harus dibayar
- Tidak ada warning tentang konsekuensi

## Target Improvement Metrics
- Reduce application abandonment rate from 40% to 15%
- Improve user understanding of loan terms from 30% to 80%
- Decrease customer support calls about status by 60%
